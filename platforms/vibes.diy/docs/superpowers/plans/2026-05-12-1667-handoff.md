## Architectural traps already discovered (avoid re-discovering)

### 1. `Apps.fileSystem` is FileSystemItem refs, not VibeFile content

Schema: `vibes.diy/api/types/types.ts:13` — `fileSystemItem` is `{ fileName, mimeType, assetId, assetURI, size, ... }`. **No content field.** Actual content lives behind `assetURI` (sql://Assets/..., s3://..., r2://...).

To get content from an `fsId`, do this — the production pattern that [`resolveVfsFromFileSystem`](vibes.diy/api/svc/intern/version-timeline.ts) and [`render-vibe.ts:55`](vibes.diy/api/svc/intern/render-vibe.ts#L55) use:

```ts
const rFetch = await vctx.storage.fetch(item.assetURI);
if (isFetchErrResult(rFetch) || isFetchNotFoundResult(rFetch)) continue;
const bytes = await stream2uint8array(rFetch.data);
const text = vctx.sthis.txt.decode(bytes);
```

**Pre-existing bug**: `loadPriorFileSystem` ([`prompt-chat-section.ts:153`](vibes.diy/api/svc/public/prompt-chat-section.ts#L153)) does NOT resolve via storage — it parses `Apps.fileSystem` as `vibeFile[]` and filters to `f.type === "code-block"`, which silently produces empty maps for all production rows. Callers (lines 536, 720, 1315) get empty vfs. This means #1690's "first-attempt CURRENT FILES" injection is effectively dead code today; the working CURRENT FILES injection comes from the recovery path's in-flight streaming resolver, not from `loadPriorFileSystem`.

**Do not fix `loadPriorFileSystem` opportunistically.** Task 13 (wire slot assembler into `assemblePromptPayload`) supersedes it. After Task 13 lands, `loadPriorFileSystem` should be evaluated for removal in Task 24 (cleanup).

### 2. `ensureApps` now takes opts, not a request envelope

Signature is `ensureApps(ctx, {env, mode, userId}, binding, fs)` after commit `4378097c`. All new callers should match. The old shape (`req: ReqWithVerifiedAuth<...>`) is gone — don't try to use it from tests or new code.

### 3. `appendTurnToChat` is the test-seed helper

Tasks 12, 13, 14, 22 in the plan reference `ctx.seedChat()` / `ctx.seedTurn()` — those helpers were imagined; they don't exist. Use `appendTurnToChat(vctx, {chatId, userId, userHandle, appSlug, fileSystem, ...})` from `vibes.diy/api/svc/intern/append-turn-to-chat.ts` instead. For seeding the first turn, `createApiTestCtx` + `ctx.createApp()` + `ctx.api.openChat()` already gets you a chat with one seeded turn (the push seed). Use `appendTurnToChat` for turn 2, 3, … N.

Test pattern (see `version-timeline.test.ts` for full example):

```ts
import { createApiTestCtx, type ApiTestCtx } from "./api-test-setup.js";
import { appendTurnToChat } from "../svc/intern/append-turn-to-chat.js";

async function userIdForSlug(ctx: ApiTestCtx, userHandle: string): Promise<string> {
  const row = await ctx.appCtx.vibesCtx.sql.db
    .select({ userId: ctx.appCtx.vibesCtx.sql.tables.userHandleBinding.userId })
    .from(ctx.appCtx.vibesCtx.sql.tables.userHandleBinding)
    .where(eq(ctx.appCtx.vibesCtx.sql.tables.userHandleBinding.userHandle, userHandle))
    .limit(1)
    .then((r) => r[0]);
  if (!row) throw new Error(`No userHandleBinding for ${userHandle}`);
  return row.userId;
}
```

## Rules-bag (READ FIRST: `agents/rules-bag.md`)

Strict rules in this codebase. Implementer subagents have already violated these and required cleanup. Pre-empt by including in dispatch prompts:

- **No `instanceof`** — use proper type narrowing. If Drizzle returns `string` from a `text` column, treat it as a string; don't `instanceof Date`-check it.
- **No `try/catch`** — use `exception2Result`.
- **No `any`** — and no `// eslint-disable-next-line @typescript-eslint/no-explicit-any` either. That's a smell.
- **Result types for I/O** — any function that touches the DB or storage returns `Promise<Result<T>>`. Pure functions don't need Result.
- **Avoid casts** — "at least 10 minutes to decide on one cast — better to ask others." If you're casting to make a signature work, the signature is probably wrong (see `ensureApps` refactor above).
- **No `export default`**, no `import X from 'y'`.
- **No mocking** — use real production code paths (that's why `appendTurnToChat` calls `ensureApps`).
- **`readonly` by default** on interface fields.
